# MQTT and Gewu Topics

The firmware connects to the China Unicom Gewu MQTT broker through ESP-12F TCP mode and sends MQTT 3.1.1 packets generated by `App/Src/mqtt.c`.

## Broker

```text
Host: dmp-mqtt.cuiot.cn
Port: 1883
TLS:  disabled
```

## Credential Format

For the current one-device-one-secret login:

```text
clientId = {deviceId}|{productKey}|{signMethod}|{authType}|{operator}
username = {deviceKey}|{productKey}
password = hmac_sha256(deviceId + deviceKey + productKey, deviceSecret)
```

For this firmware:

```text
signMethod = 0
authType   = 0
operator   = 0
```

The real password is stored only in `App/Inc/app_secrets.h`, which is ignored by Git.

## Property Report

Request topic:

```text
$sys/{productKey}/{deviceKey}/property/pub
```

Current device topic:

```text
$sys/cu1e1vp51svlk8zn/X00PdoZ4luWgnux/property/pub
```

Payload format required by Gewu:

```json
{
  "messageId": "123",
  "params": {
    "key": "smokeConcentration",
    "value": 27
  }
}
```

`smokeConcentration` is the product-model property identifier. It belongs in `params.key`, not in the MQTT topic.

Response topic:

```text
$sys/{productKey}/{deviceKey}/property/pub_reply
```

The firmware currently does not subscribe to or process `property/pub_reply`.

## Property Set Downlink

Downlink topic:

```text
$sys/{productKey}/{deviceKey}/property/set
```

Current device topic:

```text
$sys/cu1e1vp51svlk8zn/X00PdoZ4luWgnux/property/set
```

The firmware subscribes to this topic after MQTT CONNECT. The current parser does not yet decode Gewu JSON downlink payloads; it still only handles the old plain-text commands used during local MQTT testing.

## Batch Property Report

Gewu also supports:

```text
$sys/{productKey}/{deviceKey}/property/batch
$sys/{productKey}/{deviceKey}/property/batch_reply
```

The current firmware uses single-property upload only.

## MQTT.fx / Mosquitto Testing

A cloud upload can be tested from a PC with the same credentials:

```powershell
mosquitto_pub -h dmp-mqtt.cuiot.cn -p 1883 -V mqttv311 `
  -i "DEVICE_ID|PRODUCT_KEY|0|0|0" `
  -u "DEVICE_KEY|PRODUCT_KEY" `
  -P "HMAC_SHA256_PASSWORD" `
  -t '$sys/PRODUCT_KEY/DEVICE_KEY/property/pub' `
  -m '{"messageId":"123","params":{"key":"smokeConcentration","value":27}}' `
  -q 1 -d
```

Do not run a PC test client with the same client ID while the real device is connected unless you intentionally want to disconnect the device temporarily.
